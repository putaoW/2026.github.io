# 折腾GitHub Pages博客的完整记录：从问题诊断到成功修复

## 前言

今天在使用我的博客时发现了一个问题：点击主页的博客文章链接后，跳转的页面显示不正确。这让我开始了一段有趣的调试和修复之旅。通过这次折腾，我不仅解决了问题，还更深入地理解了Jekyll和GitHub Pages的工作原理。

## 问题发现

起初，我注意到博客文章的跳转有问题。点击主页上的"📝 博客文章"链接后，页面显示不正确。我需要深入代码来找出问题所在。

## 问题诊断

### 1. 检查代码结构
我首先检查了项目的整体结构，发现了两个文章目录：
- `_posts` - Jekyll的标准目录
- `post` - 旧的或补充的目录

### 2. 分析跳转逻辑
在 `index.html` 文件中，我发现这样的代码：
```html
<a href="{{ '/post/' | relative_url }}" class="nav-link">📝 博客文章</a>
```

这个链接指向了 `/post/` 路径，对应 `post/index.html` 文件。

### 3. 发现文件缺失
在检查过程中，我发现 `post` 目录被意外删除了，里面包含了一些重要的文章和资源文件。

## 解决方案

### 1. 从历史记录恢复
我使用Git命令从历史提交中恢复了被删除的 `post` 目录：
```bash
git checkout 5bbb93a~1 -- post/
```

### 2. 整合文章目录
我发现项目中存在两个文章存放位置，为了确保Jekyll能正确处理所有文章，我执行了以下操作：
- 将 `post` 目录中的文章（如"极客湾"、"某同学的文章"等）移动到 `_posts` 目录
- 这样可以确保所有文章都被Jekyll正确识别和生成

### 3. 修复资源引用
在移动文章的过程中，我发现其中一篇文章引用了图片资源：
```markdown
![游戏截图](/post/1743245307018--1896038957.jpg)
```
为了确保图片能够正常显示，我将图片文件复制到了 `otherfile` 目录，并更新了引用路径。

## 技术要点总结

### Jekyll目录结构
- `_posts`：Jekyll的标准文章目录，文件名必须遵循 `YYYY-MM-DD-文章标题.md` 格式
- `_layouts`：页面布局模板
- 其他静态资源（如图片）应放在网站根目录下的文件夹中

### 路径引用
- 使用相对路径确保在不同环境下都能正确访问资源
- 静态资源（图片、CSS、JS等）应放在根目录的子文件夹中

### Git操作
- 使用 `git checkout` 从历史提交恢复文件
- 使用 `git add`、`git commit`、`git push` 管理代码变更

## 遇到的挑战

1. **目录结构混乱**：项目中同时存在 `_posts` 和 `post` 两个文章目录，需要整合
2. **资源引用路径**：文章中引用的图片路径需要更新以确保正确显示
3. **Jekyll配置**：理解Jekyll如何处理不同目录中的文件
